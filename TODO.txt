PlaiY - Feature TODO
====================

1. Chapter Navigation
   - Extract chapter metadata from AVFormatContext->chapters in ff_demuxer.cpp
   - Add chapter list to MediaInfo and expose through C bridge
   - Add chapter list UI in settings panel with seek-on-tap
   - Effort: Low | Impact: High (MKV/MP4 files commonly have chapters)

2. Network Sources (SMB / NFS / HTTP)
   - FFmpeg avformat_open_input already supports smb://, nfs://, http:// URIs
   - Allow demuxer to accept URIs, not just local paths
   - Add "Connect to Server" dialog in library UI
   - Handle buffering states and network errors gracefully
   - Effort: Medium | Impact: High (NAS playback)

3. Playlist / Play Queue
   - Add playlist model (ordered list of file paths)
   - Implement next/previous controls in PlayerControlsView
   - Auto-advance to next file on playback completion
   - Optionally support gapless transition (pre-open next file)
   - Effort: Medium | Impact: Medium

4. Subtitle Timing Adjustment
   - Add subtitle delay adjustment (+/- offset applied to PTS)
   - Styling already implemented (font size, color, background, ASS scale)
   - Effort: Low | Impact: Medium

5. Dolby Vision and HDR10+ Tone Mapping

   - Implement per-frame dynamic metadata application for DV profiles 5, 7, 8
   - Parse HDR10+ SEI messages for dynamic tone mapping curves

   - Effort: High | Impact: High (premium HDR content)

Video Lib Pipeline
==================
Goal: Extract a standalone, reusable C++ video library from the PlaiY core
that can be embedded in other apps (e.g., real-time video chat).

1. Define Library Scope and Public API Surface
   - Audit current core/ to identify reusable vs PlaiY-specific code
   - Define what the library provides: decode, render, color conversion, audio output
   - Design a minimal C API (independent from plaiy_c.h)
   - Write an API spec document with usage examples

2. Decouple Video Decode Pipeline
   - Extract VideoToolbox and FFmpeg SW decode into a self-contained module
   - Remove dependency on PlayerEngine orchestration and internal queues
   - Accept raw packets/NAL units as input (not just demuxed AVPacket)
   - Support caller-driven threading (no internal threads by default)

3. Decouple Audio Decode and Output
   - Extract audio decoder + resampler as a standalone module
   - Separate CoreAudio output into a platform plugin interface
   - Allow callers to pull decoded PCM without using the internal ring buffer

4. Extract Metal Rendering as a Standalone Module
   - Isolate YUV-to-RGB shaders and HDR tone mapping into a reusable renderer
   - Accept CVPixelBuffer / raw frame data as input without internal frame queues
   - Expose a simple "render this frame to this MTLTexture/CAMetalLayer" API
   - Keep BT.709, BT.2020, PQ, HLG support intact

5. Build a Platform Abstraction Layer
   - Abstract Apple-specific code (VideoToolbox, CoreAudio, Metal) behind interfaces
   - Define platform backends: Apple, Linux (VAAPI/Vulkan), Windows (D3D11/Media Foundation)
   - Ensure the core library compiles on all targets without Apple frameworks

6. Design a Flexible Threading and Sync Model
   - Replace PlaiY's playback-oriented sync with pluggable sync strategies
   - Provide a low-latency mode (for real-time chat) and buffered mode (for playback)
   - Let callers own threads and drive the pipeline, or opt into managed threads

7. Add Encoding Support
   - Integrate VideoToolbox hardware encoding (H.264/HEVC)
   - Add FFmpeg software encoding fallback
   - Expose encoder configuration (bitrate, profile, latency tuning)
   - Support both file output and raw packet output (for network transport)

8. Standalone Build System and Packaging
   - Create a separate CMake project for the library (outside PlaiY's build)
   - Support building as static lib, shared lib, and framework (Apple)
   - Add vcpkg/Conan manifest for dependency management
   - Set up CI for macOS, Linux, and Windows builds

9. Test Suite and Example Apps
   - Write unit tests for each module (decode, encode, render, audio)
   - Build a minimal example app: decode a file and render to a window
   - Build a loopback example: capture -> encode -> decode -> render
   - Benchmark latency and throughput per module

10. Integrate Back into PlaiY
    - Replace PlaiY core internals with the extracted library
    - PlaiY becomes a consumer of the library (same as any other app would be)
    - Validate no regressions in playback quality, HDR, subtitles
